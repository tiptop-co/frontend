## 10. Ограничения (жёсткие)

- **Запрещено** писать комментарии в коде
- **Запрещено** использовать `useEffect` для фетчинга данных — для этого TanStack Query
- **Запрещено** передавать данные через prop drilling глубже 2 уровней — используй стор или контекст
- **Запрещено** писать бизнес-логику внутри JSX
- **Запрещено** импортировать из `mocks/` за пределами `main.tsx`
- **Запрещено** хардкодить URL в компонентах — только через константы в `shared/api/client.ts`
- **Запрещено** использовать `window.confirm`, `window.alert`, `prompt` — вместо них компонент `ConfirmDialog` из `shared/ui/ConfirmDialog.tsx` (вызывается через `await confirm(title, message)`)
- **Запрещено** хардкодить на фронте тексты, зависящие от доменной области: названия категорий меню, фильтры, статусы и прочие данные, определяемые схемой данных бекенда. Такие тексты приходят с API. Статичные UI-тексты (лейблы табов, заголовки страниц, подписи кнопок) остаются на фронте
- **Запрещено** использовать `email` как идентификатор пользователя — только номер телефона. Для ввода телефона используется компонент `PhoneInput` из `shared/ui/PhoneInput.tsx`
